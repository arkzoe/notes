# root
- 最大权限账户
- su
```shell
# switch user
su [-] [用户]
# - 切换后加载环境变量，建议加
# 用户省略，切换至root
# 切换后exit退回至上一个用户，快捷键：ctrl + d

# 普通用户切换至其他用户需密码
# root切换无需
```
- sudo
	- 为普通命令临时赋予root授权
	- 需为普通用户配置sudo认证
	- 使用root执行visudo
# 用户和用户组
- 配置多个用户/用户组
- 用户可以加入多个用户组
- 权限管控级别
	- 针对用户
	- 针对用户组
```shell
# 创建用户组
groupadd 用户组名
# 删除
groupdel 用户组名

# 创建用户
useradd [-g -d] 用户名
	# -g 指定用户的组，不指定，自动创建同名组并加入，指定-g需要组已存在
	# -d 指定用户home路径，不指定，默认：/home/用户
# 删除用户
userdel [-r] 用户名
	# -r 删除用户的home目录，不适应-r保留home
# 查看用户所属组
id [用户名]
	# 用户名不提供，查看自身
# 修改用户所属的组
usermod -aG 用户组 用户名
	# 将指定的用户加入指定的组
```
- getent
```shell
# 查看当前系统中有那些用户
getent password
用户名:密码(x):用户ID:组ID:描述信息(无用):HOME目录:执行终端(默认bash)
# 查看组
getent group
组名:组认证(x):组ID
```
# 查看权限控制
- ls -l查看内容，并显示权限细节
	- 1 表示文件、文件夹的权限控制信息
	- 权限细节![[权限细节.png]]
	- 2 表示文件、文件夹所属用户
	- 3 表示文件、文件夹所属用户组
- chmod
```shell
# 修改权限，只有文件所属用户或root能更改
chmod [-R] u=,g=,o= 文件或文件夹
# -R 对文件夹内部执行同样的操作
# 简洁写法
chmod 751 文件
0:无任何权限，即 ---
1:仅有x权限， 即 --X
2:仅有w权限， 即 -W-
3:有w和x权限，即 -WX
4:仅有r权限， 即 r--
5:有r和x权限，即 r-x
6:有r和w权限，即 rw-
7:有全部权限，即 rwx
```
- chown
```shell
# 修改文件、文件夹所属的用户、用户组
# 只有root能用
chown [-R] [用户][:][用户组] 文件或文件夹
```
