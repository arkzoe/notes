## 概述
- interface 定义接口
- jdk8 之前只能写常量和抽象方法
- 不能创建对象
- 接口时用来被类==实现（implements）==的，实现接口的类被成为==实现类，一个类可以同时实现多个接口==
```java
interface 接口名{

}
```
## 好处
- 弥补类单继承的不足，类有更多角色，功能更强大
- 可以面向接口编程吗，灵活方便切换各种业务实现，利于解耦合
## JDK8 新增的三种方法
- 默认方法（实例方法）
	- 使用default修饰，默认会public修饰
	- 只能用接口的实现类对象调用
- 私有方法（JDK9，私有实例方法）
	- 只能使用接口中的其他实例方法调用
- 静态方法
	- 默认public
	- 只能用当前接口名调用
- 增强接口能力，便于项目拓展和维护
## 注意
 1. 接口与接口可以多继承：==一个接口可以同时继承多个接口
 2. 一个接口继承多个接口，如果接口中存在方法签名冲突，则此时不支持多继承，也不支持多实现
 3. 继承父类，又实现了接口，如果有同名的默认方法，有限调用父类
	 1. 调用接口的，接口名.super.show() 中转
 4. 一个类实现了多个接口，存在同名的默认方法，可以不冲突，这个类重写该方法即可

| 类与类   | 单继承 |
| ----- | --- |
| 类与接口  | 多实现 |
| 接口与接口 | 多继承 |
## 与抽象类区别
### 相同点
1. 都是抽象形式，都可以有抽象方法，都不能创建对象
2. 都是派生子类形式：抽象类是被子类继承使用，接口是被实现类实现
3. 一个类继承抽象类，或者实现接口，都必须重写抽象方法，否则自身成为抽象类或报错
4. 都支持多态，都能实现解耦合
### 不同点
1. 抽象类可定义类的全部普通成员，接口只能定义常量，抽象方法（JDK8 新增三种方法）
2. 抽想类只能被类单继承，接口可以被类多实现
3. 一个类继承抽象类不能继承其他类，一个类实现接口（还能继承其他类或者实现其他接口）
4. 抽象类体现模板思想，更利于做父类，实现代码的复用性（最佳实践）
5. 接口更适合做功能的解耦合，解耦合性更强更灵活（最佳实践）
