- 操作的集合，将所有操作作为一个整体，同时成功或同时失败
- 默认自动提交，执行一条DML，会隐式提交事务
# 操作
- 方式一：**查看/设置事务提交方式**
```mysql
SELECT @@autocommit;
SET @@autocommit=0;
```
- 方式二：**开启事务**
```MYSQL
START TRANSACTION / BEGIN;
```
- **提交事务**
```mysql
COMMIT;
```
- **回滚**
```mysql
ROLLBACK;
```
# 四大特性
- **原子性**：事务是不可分割的最小操作单元，全部成功/失败
- **一致性**：事务完成时，使所有的数据都保持一致状态
- **隔离性**：数据库系统提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行
- **持久性**：事务提交或回滚。对数据库中的数据的改变就是永久的
# 并发事务问题
- **脏读**：一个事务读到另外一个事务还没有提交的数据
- **不可重复读**：一个事务先后读取同一条记录，两次数据不同
- **幻读**：一个事务按照条件查询数据时，没有对应数据行，插入数据时，数据又存在
